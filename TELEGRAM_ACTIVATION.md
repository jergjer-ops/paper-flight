# PAPER FLIGHT: безопасное подключение Telegram

Этот файл — чек-лист на день, когда ограничение Telegram-аккаунта будет снято. До этого момента не создавайте новые боты, не перевыпускайте токены много раз и не запускайте массовые рассылки.

## 1. Подготовить один постоянный бот

1. Проверить аккаунт через `@SpamBot` и продолжать только после снятия ограничения.
2. Выбрать одного бота и один окончательный username.
3. В BotFather перевыпустить токен один раз. Не отправлять токен в чаты, задачи, GitHub или снимки экрана.
4. Заполнить описание, аватар, политику конфиденциальности и Main Mini App.

## 2. Сначала база данных

В Supabase SQL Editor применить миграции по порядку, включая:

```text
006_game_challenges.sql
007_challenge_legacy_identity.sql
008_telegram_welcome.sql
009_close_welcome_upload.sql
010_telegram_bot_safety.sql
```

Миграция `010` закрывает подмену Telegram-посещений, добавляет ограничение частоты запросов и команду удаления данных.

## 3. Секреты Edge Functions

Задать в Supabase только через Secrets:

```text
TELEGRAM_BOT_TOKEN=<новый токен выбранного бота>
TELEGRAM_EXPECTED_USERNAME=<точный username без @>
TELEGRAM_WEBHOOK_SECRET=<случайная строка 32–64 символа>
TELEGRAM_SETUP_SECRET=<другая случайная строка 32–64 символа>
ALLOWED_ORIGINS=https://jergjer-ops.github.io
```

`TELEGRAM_EXPECTED_USERNAME` не даст случайно подключить токен другого бота. Настоящие значения нельзя сохранять в `.env.example` или Git.

## 4. Опубликовать и связать

1. Обновить `telegramBotUsername` в `public-config.js` на окончательный username.
2. Развернуть функции `telegram-flight` и `telegram-bot`.
3. Один раз вызвать служебную настройку `telegram-bot` с заголовком `x-paper-flight-setup`.
4. Убедиться, что ответ содержит ожидаемые `bot.id` и `bot.username`.
5. В BotFather указать Main Mini App URL игры.

## 5. Проверка до распространения

- `/start` присылает одно приветствие и кнопку игры.
- `/help`, `/privacy`, `/support` отвечают корректно.
- Mini App открывается только по HTTPS и показывает подтверждённый профиль.
- Старт и завершение полёта записываются, невозможный счёт отклоняется.
- Таблица лидеров и счётчик игроков загружаются.
- `/delete_me` требует явного подтверждения; тестовый профиль удаляется после `/delete_me confirm`.
- Повторная доставка одного Telegram update не создаёт дубликат.
- Частые команды и вызовы API получают ограничение, но обычная игра работает.

## 6. Безопасный запуск

Первые 24–48 часов распространять игру только среди небольшой тестовой группы. Не писать пользователям первым, не импортировать контакты и не делать автоматические приглашения. Кнопка «Бросить вызов другу» должна открываться только после явного нажатия игрока.
